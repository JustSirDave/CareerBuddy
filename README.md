# CareerBuddy

CareerBuddy is an AI-powered Telegram bot that helps job seekers create professional resumes, CVs, and cover letters through a guided conversation. It generates polished DOCX and PDF documents in minutes, with no forms or web interface required.

## Features

- **Resume generation** — guided conversational flow collects basics, experience, education, skills, and projects
- **CV generation** — academic-style multi-page CV with publications, awards, and extended experience
- **Cover letter generation** — role- and company-specific letters with AI-written body paragraphs
- **Document revamp** — upload an existing resume and get an AI-improved version
- **Three templates** — Classic, Modern, and Executive styles for resumes and CVs
- **PDF export** — convert any generated document to PDF via `/pdf`
- **AI-generated content** — skills list and professional summary generated by GPT-4o
- **Monthly usage limit** — configurable cap (default 5 documents per calendar month)
- **Inline feedback** — thumbs up/down rating after every document delivery
- **Admin tools** — `/admin stats`, `/broadcast`, `/sample` for testing
- **Message deduplication** — Redis-backed idempotency prevents double processing
- **Cloudinary storage** — generated documents survive container restarts and redeploys

## Local Development Setup

**Prerequisites:** Docker Desktop, Python 3.11+, a Telegram bot token from @BotFather.

1. Clone the repository:
   ```bash
   git clone https://github.com/JustSirDave/CareerBuddy.git
   cd CareerBuddy
   ```

2. Copy the example env file and fill in your values:
   ```bash
   cp backend/.env.example backend/.env
   # Edit backend/.env — at minimum set TELEGRAM_BOT_TOKEN and OPENAI_API_KEY
   ```

3. Start all services:
   ```bash
   docker-compose up --build
   ```

4. Run database migrations:
   ```bash
   docker compose exec api alembic upgrade head
   ```

5. Expose your local server with ngrok (Telegram requires a public HTTPS URL):
   ```bash
   ngrok http 8000
   # Copy the https URL, set PUBLIC_URL=<ngrok-url> in .env, then restart
   ```

6. The bot registers its webhook automatically on startup when `PUBLIC_URL` is a public URL. Send `/start` to your bot in Telegram to verify.

7. Run the test suite:
   ```bash
   cd backend
   python -m pytest tests/ -q
   ```

## Railway Deployment

1. Fork this repository to your GitHub account.

2. Create a new Railway project and add a service from your fork. Set the root directory to `/` — Railway reads `railway.toml` at the project root.

3. Add a **PostgreSQL** plugin and a **Redis** plugin to the project.

4. Set the following environment variables in the Railway service settings:
   - `APP_ENV=production`
   - `PUBLIC_URL` — your Railway service domain (e.g. `https://careerbuddy.up.railway.app`)
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_WEBHOOK_SECRET` — generate with `openssl rand -hex 32`
   - `OPENAI_API_KEY`
   - `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`
   - `DOWNLOAD_SECRET` — generate with `openssl rand -hex 32`
   - `DATABASE_URL` and `REDIS_URL` are injected automatically by Railway plugins

5. Deploy. Railway runs `alembic upgrade head` then starts uvicorn (see `railway.toml`).

6. The Telegram webhook is registered automatically at startup. Verify by checking Railway logs for `[BOOT] Telegram webhook set`.

7. Send `/start` to your bot. If it responds, the deployment is healthy.

## Architecture Overview

CareerBuddy is a single FastAPI process running inside a Docker container on Railway. Telegram sends updates to a webhook endpoint (`/webhooks/telegram`), where a conversation router dispatches each message to the appropriate flow (onboarding, resume, CV, cover letter, revamp, or revision). All conversation state is stored in a PostgreSQL `jobs` table as a JSON `answers` column, so the service is stateless and can be restarted at any time without losing user progress. Generated documents (DOCX and PDF) are uploaded to Cloudinary and the secure URL is stored in `job.draft_text`; the bot fetches the bytes from Cloudinary and sends them directly to the user via the Telegram Bot API. Redis provides idempotency keys to prevent duplicate message processing. APScheduler handles a monthly usage-reset cron job.
